Chapter 6

Dependencies

After you did your npm install eslint, two things happened

  1. Your package.json changed, and
  2. a new directory showed up - node_modules

Let’s talk about the first one.

If you look at package.json, you’ll see a new section called either devDependencies or dependencies (this changes if you used --save-dev or not). Dependencies are the modules that your project depends on.

As you install more and more modules, this list gets longer and longer. The funny part about this, though, is that it isn’t all of the modules that are installed.

Take a peek inside of node_modules - this is where all of your modules (aka packages!) live after they are installed. See how much stuff is hiding in there? Every folder is a single module.

Mine has approximately ten thousand things inside, even though we only installed eslint, and we only have one dependency listed!

Your code only depends on eslint, but eslint depends on other modules, and those modules depend on other modules, and it goes down and down and down until you have ten million modules in node_modules and every other programming language is rolling its eyes at you.
