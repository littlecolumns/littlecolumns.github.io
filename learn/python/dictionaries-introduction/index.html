Chapter 21

Intro to Dictionaries

This is a long chapter, so go grab a snack.

In the real world, dictionaries are things you use to look up words. Same kind of thing in Python, more or less. Let’s take a look!

Say we have a friend, and we know some stuff about her.

name = "Annie"
age = 83
town = "Cleveland"

print(name, "is", age, "and is from", town)

Nothing new, nothing crazy, totally normal.

Now, say we’re the most popular person on the planet, and we have two friends, Annie and Roberto. If we didn’t know better (which we don’t right now), we might do this:

# Annie's info
name = "Annie"
age = 83
town = "Cleveland"

# Roberto's info
name2 = "Roberto"
age2 = 103
town2 = "Reno"

# Print it all out
print(name, "is", age, "and is from", town)
print(name2, "is", age2, "and is from", town2)

It’s not the best thing I’ve ever seen, I’ll be honest. name seems like a fine variable name, but name2 and age2 and town2 just seem… tacky. Maybe I don’t like them because they aren’t very descriptive?

Now imagine if we were even more popular and had even more friends! If we had dozens of friends and kept naming variables the same way, we’d be stuck with variable names like name33 and age33 and town33 - it’d be terrible!

“A-ah!” you exclaim, ready to use your new programming knowledge. “What about lists? They’re great for storing lots of information!”

Yeah, that’s true, and it definitely makes sense. But let’s see what happens if we try to use lists:

names = ["Annie", "Roberto"]
ages = [83, 103]
town = ["Cleveland", "Reno"]

# Print out Annie's info
print(names[0], "is", ages[0], "and is from", town[0])
# Print out Roberto's info
print(names[1], "is", ages[1], "and is from", town[1])

It seems a little better, but it seems like a lot of work to use three different lists! It’s tough because a person is a single thing, but we also want to know multiple pieces of information about each of them. What we just did separates our names and ages into different variables - it splits them by topic instead of by person.

It seems like one single thing should be one single variable, yeah? In a perfect world each person could get a variable to themselves - the person has a name and an age and a town, and all of that info could live together in a single variable.

Maybe we could give each person a separate list so their stuff stays together?

annie = ["Annie", 83, "Cleveland"]
roberto = ["Roberto", 103, "Reno"]

print(annie[0], "is", annie[1], "and is from", annie[2])
print(roberto[0], "is", roberto[1], "and is from", roberto[2])

The problem now is we don’t know what makes annie[0] different than annie[1]. Which one was her name and which one was her age? I forget! At least when we used variables like names and ages we could be sure that one had names and one had ages.

If we could combine the two approaches our live swould be a lot easier. What we seem to need is one variable with multiple named things inside. Oh hey, it turns out Python can do that!

annie = {'name': 'Annie', 'age': 83, 'town': 'Cleveland'}

print(annie['name'])
print(annie['age'])
print(annie['town'])
print(annie['name'], "is", annie['age'], "and is from", annie['town'])

Hooray! Life is perfect now. Our friend annie has one variable that has information about her, organized by topic.

This data type is called a dictionary. Dictionaries are a single variable, but they can have all sorts of labeled information inside. A name can be stored as ['name'] and an age can be stored as ['age'], instead of using [0] and [1] like we had to do with lists.

They might look a little like lists, but they have a few major differences:

Dictionary List
Wrapped by Uses { and } Uses [ and ]
Inside Punctuation Has commas Has commas
Names? Yes No (numbers instead)
Creation {'name': 'Annie', 'age': 83} ['Annie', 'Roberto']
Usage annie['name'] friends[0]

Since creating a dictionary on one line looks a little messy, you can also use multiple lines. Just pay attention to the commas!

roberto = {
  'name': 'Roberto',
  'age': 103,
  'town': 'Reno'
}
print(roberto['name'], "is", roberto['age'], "and is from", roberto['town'])

Python dictionaries are pretty similar to a real dictionaries - if have a dictionary named roberto and want to look up his name, I open up to the 'name' page. If I want to learn about his age, I find the definition under 'age'. With Python, it just becomes roberto['name'] and roberto['age']

We’ve been using dictionaries to hold multiple pieces of information about the same thing, but we can also use them for the same piece of information about multiple things1.


population = {
  'Santiago': 7.2,
  'San Diego': 1.407,
  'Nara': 1.401,
  'Moscow': 12.19
}
print("The population of Moscow is", population['Moscow'])



I find it’s a lot more convenient than a list, because


    

  • You get to store everything in one variable, and
    • 

  • You get to name each of your data points
    • 



The one difference between Python dictionaries and real dictionaries is you can’t have duplicates in a Python dictionary. Only one definition for Moscow! Only one definition for Nara! But we’ll be okay.
